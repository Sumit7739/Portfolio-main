Hisab-Kitab - Overview

Hisab-Kitab is a versatile financial management application designed to help users record and manage their daily transactions, whether tracking personal expenses or managing transactions with others. The app is accessible via both web and mobile platforms, ensuring convenience for users across multiple devices.

The application offers features such as transaction recording, separate entry categories for personal and group expenses, cross-device access, and user connections for shared financial management. These functionalities aim to simplify expense tracking and promote organized financial activities.

In the 2.3 update, a new feature was introduced to support storing client-related information for LIC or other insurance purposes. This allows users to log and organize party-specific insurance details along with related financial transactions â€” adding another layer of utility for professionals managing such accounts.

Key Features

Hisab-Kitab offers a range of features designed to provide a seamless financial management experience:

Role-Based Access

Hisab-Kitab supports two primary user roles: Regular User and Agent. Agents are required to enter their agency code during signup, unlocking a specialized dashboard that enables them to manage and maintain client-related insurance records efficiently. This system empowers agents to handle their LIC or insurance-related data separately from general transactions.

Transaction Recording

Users can record individual transactions as Debit (expenses) or Credit (income), adding details such as amount, purpose, and associated person.

Separate Entry Categories

Allows users to track personal daily expenses separately from transactions involving other individuals, promoting organized financial management.

Cross-Device Access

The app is accessible seamlessly on web and mobile platforms with a unified account, ensuring users can manage their finances on the go.

User Connections

Users can connect with others to share or manage transaction records, granting permissions for either viewing only or adding transactions.
